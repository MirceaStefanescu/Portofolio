# Contributing

Thanks for taking the time to contribute.

## Development workflow
- Create a feature branch.
- Run `make lint` and `make test` before opening a PR.
- Keep changes focused and update docs when behavior changes.

## Commit messages
- Use short, descriptive messages (e.g., "Add order retry handling").

## Code of conduct
This project follows the Code of Conduct in `CODE_OF_CONDUCT.md`.
